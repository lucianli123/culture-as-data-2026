# A pip constraints file caps the few transitive dependencies that, left free, drift
# past the versions tested against Colab's image and break at import time (the numpy
# fault line in particular). Install with:
#     %pip install -q -r requirements.txt -c constraints.txt
#
# A constraint does NOT install a package. It only bounds the version IF something
# pulls it in. So leaving torch/numpy here is safe: if Colab already satisfies them,
# nothing is reinstalled; if a fresh container needs them, these are the tested ceilings.
#
# Re-freeze these from a successful Colab `pip freeze` and update the date below.
# tested-as-of: PLACEHOLDER, replace with the date of the first green Colab smoke test.
